## Design System Rules (MANDATORY)
- Semantic tokens ONLY: `bg-background`, `text-foreground`, `bg-primary` — NEVER raw Tailwind colors
- Dark mode: `dark:` variants on everything that has a background/text color
- Mobile-first: default = 375px, add `md:` and `lg:` for larger screens
- Touch targets: 44×44px minimum (WCAG 2.2)
- All copy from `src/config/copy/` — never hardcode strings in JSX
- Icons: Lucide only, nav=24px inline=20px status=16px
- Animations: `duration-150` micro, `duration-200` fast, `duration-300` normal

## Component Pattern
Server Component page → Client Component wrapper → Sub-components
```
src/app/dashboard/[role]/feature/page.tsx (Server)
src/app/dashboard/[role]/feature/feature-page-client.tsx (Client, 'use client')
src/components/[role]/FeatureCard.tsx (pure presentational)
```

- Every form: Zod schema + react-hook-form, inline errors (not toast)
- WCAG 2.1 AA contrast minimum
- No horizontal scroll at ANY viewport width

## Pages to create

### src/app/(auth)/login/page.tsx
- Server Component — if session exists, redirect to /dashboard/[role] (role from profiles table)
- Renders <LoginPageClient />

### src/app/(auth)/login/login-page-client.tsx
'use client' — email + password form using react-hook-form + Zod:
- Zod schema: email (z.string().email()), password (z.string().min(8))
- On submit: call supabase.auth.signInWithPassword({ email, password })
- On success: router.push('/dashboard/' + profile.role)
- On error: show inline error message (not toast)
- Link to /signup and /forgot-password

### src/app/(auth)/signup/page.tsx + signup-page-client.tsx
- Fields: email, password, confirmPassword, role (select: student/sponsor/university/agent/partner)
- On submit: supabase.auth.signUp → then insert into profiles table via tRPC publicProcedure
- After signup: show "Check your email to confirm" message

### src/app/(auth)/forgot-password/page.tsx + forgot-password-page-client.tsx
- Field: email only
- On submit: supabase.auth.resetPasswordForEmail({ email, redirectTo: '/update-password' })
- Show success state: "Reset link sent to [email]"

### src/app/(auth)/update-password/page.tsx + update-password-page-client.tsx
- Requires active session (Supabase sends magic link that creates session)
- Fields: newPassword, confirmPassword
- On submit: supabase.auth.updateUser({ password: newPassword })
- On success: redirect to /dashboard/[role]

### src/app/(auth)/layout.tsx
- Centered layout, max-w-md, vertical center
- Brand logo at top (use next/image, src="/brand/logos/logo.svg")
- Dark mode support

## tRPC procedure needed

In src/server/routers/student.ts (or a shared auth router), add:

```typescript
createProfile: publicProcedure
  .input(z.object({
    userId: z.string().uuid(),
    role: z.enum(['student', 'sponsor', 'university', 'admin', 'agent', 'partner']),
  }))
  .mutation(async ({ ctx, input }) => {
    await ctx.db.insert(profiles).values({
      userId: input.userId,
      role: input.role,
    });
  }),
```
