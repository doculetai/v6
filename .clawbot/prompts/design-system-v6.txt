Build V6's design system from scratch — it must BEAT V5's. Not just port CSS, but architect a proper theme system that's easy to manage, administer, and extend for white-label partners.

## What V5 Had (baseline to beat)
V5 had a 1484-line globals.css with Obsidian Blues palette. It works but:
- Hard to customize per partner (require touching raw CSS)
- Dark mode tokens scattered across 3 `.dark {}` blocks
- No programmatic control over theme

## What V6 Needs: A TypeScript-First Theme System

### Architecture

```
src/config/
  themes/
    types.ts          — Theme interface (full token shape)
    doculet.ts        — Default Doculet theme (Obsidian Blues, premium fintech)
    index.ts          — Theme registry + active theme resolver
  theme.ts            — Re-export: import { theme } from "@/config/theme"

src/lib/
  theme-to-css.ts     — Converts Theme object → CSS custom properties string

src/components/
  theme-provider.tsx  — Client component: applies theme via CSS vars + data-theme attribute
```

### Theme Interface (src/config/themes/types.ts)

```typescript
export interface ColorScale {
  50: string; 100: string; 200: string; 300: string; 400: string;
  500: string; 600: string; 700: string; 800: string; 900: string; 950: string;
}

export interface SemanticTokens {
  background: string;
  foreground: string;
  card: string;
  cardForeground: string;
  primary: string;
  primaryForeground: string;
  secondary: string;
  secondaryForeground: string;
  muted: string;
  mutedForeground: string;
  accent: string;
  accentForeground: string;
  destructive: string;
  border: string;
  input: string;
  ring: string;
  success: string;
  warning: string;
}

export interface Theme {
  name: string;
  displayName: string;
  description: string;

  // Color palettes
  colors: {
    primary: ColorScale;    // Obsidian Blues (or partner brand)
    neutral: ColorScale;    // Cool slate
    success: Partial<ColorScale>;
    warning: Partial<ColorScale>;
    destructive: Partial<ColorScale>;
    accent: Partial<ColorScale>;
  };

  // Semantic tokens (light mode)
  light: SemanticTokens;

  // Semantic tokens (dark mode)
  dark: SemanticTokens;

  // Typography
  fonts: {
    sans: string;       // CSS font-family value
    serif: string;
    mono: string;
  };

  // Scale
  radius: {
    sm: string; md: string; lg: string; xl: string; '2xl': string; full: string;
  };

  shadows: {
    xs: string; sm: string; md: string; lg: string; xl: string; '2xl': string;
  };

  // Motion
  duration: {
    micro: string; fast: string; normal: string; slow: string; slower: string;
  };

  // Layout
  layout: {
    sidebarWidth: string;
    headerHeight: string;
    contentMaxWidth: string;
  };
}
```

### Theme Registry (src/config/themes/index.ts)

```typescript
import { doculetTheme } from './doculet';
import type { Theme } from './types';

export const themes: Record<string, Theme> = {
  doculet: doculetTheme,
};

export function getTheme(name: string): Theme {
  return themes[name] ?? themes.doculet;
}

export const defaultTheme = doculetTheme;
```

### ThemeProvider (src/components/theme-provider.tsx)

Client component that:
1. Generates CSS string from the active theme using theme-to-css.ts
2. Injects it as a `<style>` tag in `<head>` via useEffect
3. Manages light/dark mode toggle (stores preference in localStorage + sets .dark class on html)
4. Exposes `useTheme()` hook
